# Prompt: Arquitetura Event-Driven

## Contexto
[Descreva o fluxo de negócio que precisa ser orientado a eventos]

## Stack disponível
[ex: Kafka, RabbitMQ, AWS SNS/SQS, Azure Service Bus]

## Tarefa
1. Proponha o design de eventos (nome, payload, produtor, consumidor).
2. Defina a topologia de tópicos/filas com estratégia de particionamento.
3. Trate os seguintes cenários: retry, dead-letter queue, idempotência.
4. Mostre um exemplo de schema de evento com versionamento.
5. Indique padrões como Saga ou Outbox quando aplicável.
