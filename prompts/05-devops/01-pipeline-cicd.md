# Prompt: Modelagem de Pipeline CI/CD

## Contexto
- **Repositório:** [mono-repo / multi-repo]
- **Linguagem/Framework:** [ex: Node.js, Java Spring, Python]
- **Plataforma de CI:** [ex: GitHub Actions, GitLab CI, CircleCI]
- **Ambiente de deploy:** [ex: AWS ECS, Kubernetes, Heroku]
- **Ambientes:** [ex: dev, staging, production]

## Tarefa
1. Proponha as etapas do pipeline (lint, test, build, scan, deploy).
2. Gere o arquivo de configuração do pipeline para a plataforma escolhida.
3. Inclua estratégias de deploy: blue/green, canary ou rolling update.
4. Adicione gates de qualidade (cobertura mínima, SAST, etc.).
5. Sugira estratégias de rollback automatizado.
